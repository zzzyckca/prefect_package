Prefect 3.x Local Architecture

A resilient, open-source orchestration system replacing legacy Windows Task Scheduler.

System Flow Overview

1. Configuration

prefect.yaml definitions push directly to the server via deployment.bat.

2. Prefect Server

Background services orchestrated by start.bat.

UI Dashboard Accessible

3. Master Pipeline

Pure Python execution of operational scripts in isolated environments.

Enterprise Capabilities

True Business Day AI

Unlike Task Scheduler, our pipeline uses Pandas and Python to accurately calculate the Nth Toronto business day, automatically skipping weekends and statutory holidays dynamically.

Auto-Email Reporting

System crashes instantly dispatch diagnostic emails to stakeholders using the `prefect-email` framework, eliminating the "silent failures" associated with legacy cron jobs.

Isolated Environments

The entire architecture is self-contained in C:\prefect. The `install.bat` and `config.ini` files mean the system can be destroyed and rebuilt in seconds without breaking the host machine.

Adding a New Job (Manager view)

You don't need to know Prefect to schedule a script. Simply copy your code to the server, and tell the YAML configuration to trigger it.

prefect.yaml
- name: "monthly-report"
  entrypoint: "master_pipeline.py:run_script_list"
  parameters:
    target_business_day: 5 # Runs mathematically on true business days
    scripts_to_run:
      - 'c:\temp\your_new_script.py'
  schedules:
    - cron: "0 9 * * *"
Requires 0 lines of Prefect Python code.
